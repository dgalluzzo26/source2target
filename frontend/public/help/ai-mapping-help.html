How Smart Mapper's AI generates mapping suggestions

üéØ What is AI Discovery?

AI Discovery is the core feature of Smart Mapper V4. It automatically analyzes your source fields and generates mapping suggestions for entire target tables at once.

For each target column, AI Discovery:

  1. Searches for historical mapping patterns from past projects
  2. Uses vector search to find semantically similar source columns
  3. Rewrites SQL expressions using your specific source columns
  4. Calculates confidence scores for each suggestion

‚ö° How Discovery Works

1

Pattern Lookup

AI searches mapped_fields for historical SQL patterns that match the target column. These patterns come from previous mappings on other projects.

2

Vector Search

Using semantic similarity, AI finds source columns that match the columns needed in the pattern. It considers column names, descriptions, and data types.

3

SQL Rewriting (LLM)

The LLM rewrites the historical SQL pattern, replacing original table/column names with your source tables and columns while preserving JOINs and transformations.

4

Confidence Scoring

Based on semantic similarity matches and SQL complexity, each suggestion gets a confidence score (0-100%) to help you prioritize review.

üîß Special Case Patterns

Some patterns are automatically detected and handled specially:

TypeDetectionHow It's Handled
AUTO_MAPPED Pattern SQL is "Auto Generated" No SQL needed - field is auto-populated (e.g., identity columns). Just approve.
HARDCODED Pattern SQL starts with "Hard coded as" SQL generated as SELECT 'value' AS column. Review the constant value.
NOT_APPLICABLE Pattern SQL is "N/A" Field not needed in this migration. Review and skip if appropriate.

üí° Tips for Better Results

üìù Source Field Descriptions Matter!

The AI relies heavily on the src_comments column in your uploaded source fields. Clear, detailed descriptions lead to much better matches.

üéØ Quality Over Speed

Take time to review suggestions carefully. Approved mappings become patterns for future projects - good patterns lead to better AI suggestions.

‚ö†Ô∏è Silver Tables Are Constant

Tables with "silver" in their path are treated as constant reference tables. The AI won't try to replace columns from silver tables - only bronze/source tables.
