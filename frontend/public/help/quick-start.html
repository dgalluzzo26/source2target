üéØ Choose Target Field

Select or let AI suggest the target field:

  • AI Suggestions (Recommended): Click Get AI Suggestions to see AI-recommended target fields based on semantic similarity
  • Manual Selection: Browse the semantic table and manually select a target field
  • Review the suggested target field's table name, column name, data type, and description
  • Accept the suggestion or modify it
ü§ñ AI Power: The AI analyzes field names, descriptions, and data types to suggest the best target matches. It's especially good at detecting multi-field patterns like "full_name" for FIRST_NAME + LAST_NAME.
3

üîÑ Configure Transformations

Apply SQL transformations to each source field:

  • For each selected source field, choose a transformation from the dropdown:
    • TRIM + INITCAP - Capitalize first letter of each word, remove spaces
    • TRIM + UPPER - Convert to uppercase, remove spaces
    • TRIM + LOWER - Convert to lowercase, remove spaces
    • TO_DATE - Convert string to date format
    • No Transformation - Use the field as-is
  • Transformations are applied before concatenation
  • Custom transformations can be managed in Administration ‚Üí Transformation Management
4

üîó Set Concatenation Strategy

Define how multiple source fields are combined:

  • SPACE - Combine with space: "John" + "Doe" ‚Üí "John Doe"
  • COMMA - Combine with comma: "John" + "Doe" ‚Üí "John,Doe"
  • PIPE - Combine with pipe: "John" + "Doe" ‚Üí "John|Doe"
  • CUSTOM - Specify your own separator (e.g., " - ", ", ")
  • NONE - No concatenation (for single-field mappings)
üí° Tip: Choose SPACE for names, COMMA for address components, PIPE for data exports, or CUSTOM for special formatting needs.
5

üîÄ Define Join Conditions (Optional)

If mapping fields from multiple source tables, define how they join:

  • Click Add Join Condition to create a join
  • Specify:
    • Left Table & Column - First join key
    • Join Type - INNER, LEFT, RIGHT, FULL OUTER
    • Join Operator - = (equals), != (not equals), <, >, etc.
    • Right Table & Column - Second join key
  • Add multiple join conditions if needed (joined with AND)
üí° Tip: Most joins use INNER with = operator. Only needed when combining fields from different source tables.
6

üíæ Review & Save Mapping

Preview the SQL transformation and save:

  • Review the Transformation Expression Preview showing the generated SQL
  • Verify:
    • Source fields are in the correct order
    • Transformations are applied correctly
    • Concatenation strategy is appropriate
    • Join conditions (if any) are accurate
  • Click Save Mapping to commit the mapping
  • The mapping appears in the Mapped Fields tab
‚úÖ Success! Your mapping is now saved and can be exported for use in ETL processes. You can edit transformations, joins, and concatenation strategy later, but cannot change the target field or add/remove source fields without creating a new mapping.

‚ùì Common Questions

Q: Can I map multiple source fields to one target field?

A: Yes! Select multiple source fields (e.g., FIRST_NAME + LAST_NAME) and map them to a single target field (e.g., full_name). Choose a concatenation strategy to combine them.

Q: Can I edit a mapping after saving it?

A: Yes! You can edit transformations, join conditions, and concatenation strategy. However, you cannot change the target field or add/remove source fields - those require creating a new mapping.

Q: What transformations are available?

A: System transformations include TRIM, INITCAP, UPPER, LOWER, and TO_DATE. Admins can add custom transformations in Administration ‚Üí Transformation Management.

Q: When do I need join conditions?

A: Only when mapping fields from multiple source tables. For example, combining T_MEMBER.FIRST_NAME and T_ADDRESS.CITY requires a join condition linking the tables.

Q: How do I export my mappings?

A: Go to the Mapped Fields tab and click the Export Mappings button. You'll get a CSV with all mapping details and generated SQL expressions.

Q: Can I bulk upload unmapped fields?

A: Yes! Use the Upload Template button in Unmapped Fields to bulk add source fields via CSV.
