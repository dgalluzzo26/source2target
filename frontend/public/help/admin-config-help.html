üë§ Admin Access

Only users who are members of designated admin groups can access configuration settings and semantic table management.

How to Check Admin Status

  • Look at the badge next to your name in the header
  • If it says "Admin" (green badge), you have admin access
  • If it says "User" (grey badge), you only have user-level access
  • The Configuration menu item only appears for admins

Adding Admin Users

To grant admin access to a user:

  1. Add the user to one of the admin groups in Databricks workspace
  2. Groups are configured in the admin_groups setting
  3. User will see admin badge and menu items on next login

üîç Vector Search Configuration

Configure AI-powered semantic search for finding similar fields by meaning.

Setting Description Notes
endpoint_name Vector Search endpoint name Must be created in Databricks
index_name Index containing embedded fields Must be synced with semantic table
embedding_column Column used for semantic queries Usually "semantic_field"
max_results Max candidate matches to return Default: 25 (recommended)

Vector Search Prerequisites

  • Endpoint: Must be created and online in Databricks
  • Index: Must be created from semantic table with embeddings
  • Sync: Index must be synced when semantic table changes
  • Status: Check "Vector Search: Available" on Introduction page
üí° Pro Tip: If vector search is slow or timing out, try reducing max_results from 25 to 15.

ü§ñ AI Model Configuration

Configure the LLM endpoint that analyzes matches and generates suggestions.

Setting Description Recommended Value
endpoint_name Model Serving endpoint name databricks-dbrx-instruct-serving
max_ai_results Number of suggestions to generate 10
temperature Model creativity (0.0 - 1.0) 0.1 (focused, not creative)
max_tokens Maximum response length 2000

Model Endpoint Requirements

  • Status: Endpoint must be in "Ready" state
  • Access: App must have permission to query endpoint
  • Cold Start: First request may take 30-60 seconds
  • Timeout: Requests timeout after 60 seconds
‚ö†Ô∏è Cost Warning: AI model calls consume compute resources. Monitor usage in Databricks to control costs.

üìä Managing Semantic Table

The semantic table contains all target field definitions that AI uses for matching.

Semantic Table Operations

  • View: Browse all target fields in the Semantic Table tab (admin only)
  • Search: Filter by table name, column name, or description
  • Edit: Update target field details, datatypes, nullable, comments
  • Delete: Remove target fields (with confirmation)
  • Bulk Add: Import multiple target fields via CSV (coming soon)

Required Columns

Column Purpose
tgt_table_name Target table logical name
tgt_column_name Target column logical name
tgt_table_physical_name Actual table name in database
tgt_column_physical_name Actual column name in database
tgt_comments Description (helps AI matching)
tgt_physical_datatype Column datatype (STRING, INT, etc.)
tgt_nullable "Null" or "Not Null"
semantic_field Combined text for vector search
üí° Important: After adding/editing target fields, the Vector Search index must be resynced to include the changes.

‚úÖ Configuration Validation

Always validate configuration before saving to prevent system errors.

Validation Checklist

  • ‚úÖ Database: SQL Warehouse is running and accessible
  • ‚úÖ Tables: Mapping and semantic tables exist in specified catalog/schema
  • ‚úÖ Vector Search: Endpoint is online and index exists
  • ‚úÖ AI Model: Model endpoint is in "Ready" state
  • ‚úÖ Permissions: App has access to all resources
  • ‚úÖ Admin Groups: At least one valid Databricks group listed

Testing Configuration

  1. Save your configuration changes
  2. Go to the Introduction page
  3. Check all four system status indicators:
    • Database Connection - should be "Connected"
    • Vector Search - should be "Available"
    • AI Model - should be "Ready"
    • Configuration - should be "Valid"
  4. If any show errors, review and fix configuration

üîß Troubleshooting

Database Connection Fails

  • Verify SQL Warehouse is running (not stopped)
  • Check server hostname and HTTP path are correct
  • Ensure app has OAuth access to workspace
  • Verify catalog and schema names exist

Vector Search Not Available

  • Check endpoint is created and online in Databricks
  • Verify index name matches exactly
  • Ensure index is synced with semantic table
  • Check app has permission to query endpoint

AI Model Not Ready

  • Verify model endpoint exists and is serving
  • Check endpoint status in Databricks (should be "Ready")
  • Wait for cold start if first request (30-60 seconds)
  • Ensure app has permission to query endpoint

Users Not Showing as Admin

  • Verify user is member of one of the configured admin groups
  • Check group names are spelled correctly (case-sensitive)
  • User may need to log out and back in
  • Check Databricks workspace for group membership
