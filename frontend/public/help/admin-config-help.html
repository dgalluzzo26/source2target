üë§ Admin Access

Only users who are members of designated admin groups can access configuration settings and semantic table management.

How to Check Admin Status

  • Look at the badge next to your name in the header
  • If it says "Admin" (green badge), you have admin access
  • If it says "User" (grey badge), you only have user-level access
  • The Configuration menu item only appears for admins

Adding Admin Users

To grant admin access to a user:

  1. Add the user to one of the admin groups in Databricks workspace
  2. Groups are configured in the admin_groups setting
  3. User will see admin badge and menu items on next login

üîó V2 Multi-Field Mapping Features

Version 2 introduces powerful multi-field mapping capabilities for complex ETL scenarios.

Key Capabilities

  • Multiple Source Fields: Map 2+ source fields to one target field (e.g., FIRST_NAME + LAST_NAME ‚Üí full_name)
  • Individual Transformations: Apply different SQL transformations to each source field
  • Concatenation Strategies: Combine fields with SPACE, COMMA, PIPE, CUSTOM separator, or NONE
  • Cross-Table Joins: Map fields from different source tables with SQL join conditions
  • Generated SQL: Complete transformation expression auto-generated for ETL use
  • Export Mappings: Download all mappings as CSV with SQL expressions

Editing Restrictions

To maintain data integrity, certain mapping changes are restricted:

Can Edit Cannot Edit
  • Transformations on source fields
  • Concatenation strategy
  • Custom separator
  • Join conditions
  • Target field (table/column)
  • Adding/removing source fields
  • Changing source field order
üí° Tip: To change the target field or source field structure, delete the existing mapping and create a new one.

üõ°Ô∏è Transformation Management

Manage SQL transformation templates that can be applied to source fields during mapping. Navigate to Administration ‚Üí Transformation Management.

Transformation Library

The transformation library contains reusable SQL expressions that standardize data during ETL:

  • TRIM - Remove leading/trailing whitespace: TRIM(field)
  • TRIM + INITCAP - Capitalize first letter of each word: INITCAP(TRIM(field))
  • TRIM + UPPER - Convert to uppercase: UPPER(TRIM(field))
  • TRIM + LOWER - Convert to lowercase: LOWER(TRIM(field))
  • TO_DATE - Convert string to date: TO_DATE(field, 'yyyy-MM-dd')
  • Custom Transformations - Add your own SQL expressions

Managing Transformations

Operation Description
Add Transformation Click "Add Transformation" button, provide code, description, and SQL expression template
Edit Transformation Click edit icon to modify description or SQL expression (custom transformations only)
Delete Transformation Click delete icon to remove transformation (custom transformations only)
Search Use search box to filter transformations by code or description

System vs Custom Transformations

  • System Transformations: Built-in transformations marked with a shield icon üõ°Ô∏è - cannot be edited or deleted
  • Custom Transformations: User-created transformations that can be fully managed

Creating Custom Transformations

  1. Click Add Transformation
  2. Enter a unique Code (e.g., "CUSTOM_FORMAT")
  3. Provide a Description (e.g., "Format phone number")
  4. Enter the SQL Expression using {field} as placeholder:
    • Example: REGEXP_REPLACE({field}, '[^0-9]', '')
    • The {field} placeholder will be replaced with actual field names
  5. Click Save
  6. New transformation appears immediately in the mapping interface
‚ö†Ô∏è Important: Test custom SQL expressions thoroughly before deploying. Invalid SQL will cause mapping errors.

üìä Semantic Management

Manage target field definitions in the semantic table. Navigate to Administration ‚Üí Semantic Management.

About the Semantic Table

The semantic table contains all target field definitions that AI uses for matching source fields to target fields. Each record represents one target column available for mapping.

Semantic Table Operations

  • View: Browse all target fields in the Semantic Management view (admin only)
  • Search: Filter by table name, column name, or description
  • Add: Create new target field definitions manually
  • Edit: Update target field details, datatypes, nullable, comments
  • Delete: Remove target fields (with confirmation)
  • Vector Search Sync: Automatically syncs vector search index after changes

Semantic Table Schema

Column Purpose Example
tgt_table_name Target table logical name slv_member
tgt_column_name Target column logical name full_name
tgt_table_physical_name Actual table name in database healthcare_data.silver.member
tgt_column_physical_name Actual column name in database member_full_name
tgt_comments Description (helps AI matching) Member full name combined from first and last name
tgt_physical_datatype Column datatype STRING, INT, DATE, TIMESTAMP, etc.
tgt_nullable Null constraint "Null" or "Not Null"
semantic_field Combined text for vector search Auto-generated from table, column, and comments

Vector Search Sync

The vector search index must stay synchronized with the semantic table for AI suggestions to work correctly:

  • Automatic Sync: The system automatically triggers a sync after add/edit/delete operations
  • Sync Status: Look for "Vector search index sync triggered successfully" message after changes
  • Sync Delay: Changes may take 1-2 minutes to appear in AI suggestions
  • Manual Sync: If needed, contact your Databricks admin to manually sync the index
üí° Important: After adding/editing target fields, wait 1-2 minutes before expecting them to appear in AI suggestions. The vector search index syncs asynchronously.

‚úÖ Configuration Validation

Always validate configuration before saving to prevent system errors.

Validation Checklist

  • ‚úÖ Database: SQL Warehouse is running and accessible
  • ‚úÖ Tables: Mapping and semantic tables exist in specified catalog/schema
  • ‚úÖ Vector Search: Endpoint is online and index exists
  • ‚úÖ AI Model: Model endpoint is in "Ready" state
  • ‚úÖ Permissions: App has access to all resources
  • ‚úÖ Admin Groups: At least one valid Databricks group listed

Testing Configuration

  1. Save your configuration changes
  2. Go to the Introduction page
  3. Check all four system status indicators:
    • Database Connection - should be "Connected"
    • Vector Search - should be "Available"
    • AI Model - should be "Ready"
    • Configuration - should be "Valid"
  4. If any show errors, review and fix configuration

üîß Troubleshooting

Database Connection Fails

  • Verify SQL Warehouse is running (not stopped)
  • Check server hostname and HTTP path are correct
  • Ensure app has OAuth access to workspace
  • Verify catalog and schema names exist
  • Confirm all V2 tables exist (mapped_fields, mapping_details, mapping_joins, transformations, unmapped_fields)

Vector Search Not Available

  • Check endpoint is created and online in Databricks
  • Verify index name matches exactly
  • Ensure index is synced with semantic table
  • Check app has permission to query endpoint
  • Wait 1-2 minutes after semantic table changes for sync to complete

AI Model Not Ready

  • Verify model endpoint exists and is serving
  • Check endpoint status in Databricks (should be "Ready")
  • Wait for cold start if first request (30-60 seconds)
  • Ensure app has permission to query endpoint

Users Not Showing as Admin

  • Verify user is member of one of the configured admin groups
  • Check group names are spelled correctly (case-sensitive)
  • User may need to log out and back in
  • Check Databricks workspace for group membership

Transformations Not Appearing

  • Verify transformation was saved successfully (check for success message)
  • Refresh the mapping page if transformations don't appear in dropdowns
  • Check for duplicate transformation codes (must be unique)
  • Ensure transformations table exists in database

Mapping Edit Fails

  • Cannot edit target field - this requires deleting and recreating the mapping
  • Cannot add/remove source fields - create a new mapping instead
  • Check that transformation expressions are valid SQL
  • Verify join conditions reference existing tables and columns
  • Ensure custom separators are valid strings

Export Fails or Missing Data

  • Ensure you have at least one mapped field to export
  • Check browser allows CSV downloads
  • Verify database connection is active
  • Check that mapped_fields and mapping_details tables are accessible

Vector Search Index Not Syncing

  • Check the browser console for error messages
  • Verify the vector search endpoint has permission to sync
  • Ensure the index source table matches the semantic table
  • Try manually triggering a sync via Databricks console
  • Check Databricks logs for vector search sync errors
