Executive Summary

Smart Mapper uses a Hybrid AI approach combining vector search and LLM reasoning to intelligently suggest data field mappings from source systems to target schemas. The system learns from historical patterns and user feedback, with each mapping stored as a single row with a complete SQL expression for maximum flexibility.

üéØ Key Innovation: Single Row = Complete Mapping

Instead of complex multi-table relationships, each mapping is stored as one row with a SQL expression that captures all transformations, joins, and unions. This simplifies the data model and makes AI learning more effective.

The Problem We Solve

üìã Challenge: Names Don't Match, But Descriptions Do

Source table/column names are completely different from target names, but column descriptions ARE similar. We need to:

  • Match source to target using description similarity
  • Learn transformations (TRIM, UPPER) from historical patterns
  • Detect single vs multi-field mappings
  • Suggest complete SQL expressions
  • Avoid previously rejected mappings
Example:
Source: patient_data.mbr_ssn_num ("Social security number of the patient")
Target: Member.SSN ("Member social security number")

Names don't match, but descriptions are semantically similar!

Pattern Aggregation & Frequency Boosting

When multiple clients have mapped similar source fields to the same target, the system aggregates patterns and applies frequency-based boosting to prioritize well-validated mappings.

1

Pattern Grouping

Historical patterns are grouped by target field + column count

Grouping Criteria (what makes patterns "the same"):
‚úì Same target field (e.g., MBR_FNDTN.FIRST_NM)
‚úì Same source column count (1 col vs 2 cols = different patterns)

NOT required to match:
‚úó Source column names (can differ across clients)
‚úó Source table names (can differ)
‚úó Exact transformation syntax (we vote on this)
‚Üì
2

Patterns grouped into 2 distinct options, ranked by frequency

Frequency Boost Formula

More historical patterns = higher confidence = bigger boost

Formula: boost = 1.5 √ó (1 + min(1.0, log‚ÇÇ(count) √ó 0.3))

5

Transformation Voting

Within each group, transformations are voted on individually

Voting Rules:
‚Ä¢ Each transformation is voted on independently
‚Ä¢ Include if it appears in >50% of patterns in the group
‚Ä¢ Order: TRIM first, then case transforms (UPPER, LOWER, PROPER_CASE)

Example (3 single-field patterns):
‚Ä¢ TRIM appears in 3/3 ‚Üí ‚úì INCLUDED
‚Ä¢ PROPER_CASE appears in 2/3 ‚Üí ‚úì INCLUDED
‚Ä¢ UPPER appears in 0/3 ‚Üí ‚úó NOT INCLUDED

Winning Expression: PROPER_CASE(TRIM(source_column))

üéØ Why This Matters

As more clients are onboarded and more mappings are created, the AI becomes increasingly confident about common patterns. A mapping validated by 5 different clients is much more reliable than one from a single client. The frequency boost ensures these well-validated patterns rise to the top of suggestions.
